# BRAT plugin update — per-plugin refresh button vs global Check-for-updates

**Reference:** В Obsidian, для force-update beta-plugin до latest GitHub release **используй per-plugin circular-arrow refresh button** в `Settings → BRAT → Beta plugin list`. Command Palette `BRAT: Plugins: Check for updates to all beta plugins and UPDATE` может показать notice "Checking for plugin updates COMPLETED" БЕЗ реального download — empirically наблюдалось 2026-05-17.

**Why prefer per-plugin refresh:**
- Forces full re-download от release tag, не consultee локальный version cache
- Notice сразу показывает старая→новая version: "Plugin has been updated from version X to Y"
- Если versions совпадают — explicit "Plugin is up to date" notice (vs silent COMPLETED)

**How to trigger:**

1. `Cmd+,` (Settings) → Community plugins panel слева → click `BRAT`
2. Scroll down до `Beta plugin list`
3. Найти plugin entry (e.g., `kitelev/exocortex-obsidian-plugin`)
4. Click ↻ (refresh, leftmost из 3 buttons)
5. Watch для notice: "Plugin has been updated from version X to Y"
6. Reload Obsidian (`Cmd+P` → "Reload app without saving") to pick up new code in memory
