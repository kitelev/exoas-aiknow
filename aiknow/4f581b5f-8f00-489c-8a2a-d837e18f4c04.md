## Rule
Когда нужно verify состояние exocortex-plugin warnings (count, types, files) — **не полагаться на DevTools console**, идти напрямую в файл лога.

## Why
Plugin использует свой `fileLogChannel` (~/.obsidian/plugins/exocortex/logs via `this.logger.warn(...)`) который пишет в `/Users/kitelev/vault-2025/exocortex-logs.txt` (на момент 2026-05-17 ~6 MB, 52k+ строк). Console default level (Info+Warnings+Errors) **этих warning'ов не показывает** — потому что они эмитятся через custom logger pipeline, не через native console.warn. Я столкнулся с false-positive: DevTools filter "Skipping" → 0 matches → confidently сказал "ни одного warning". User push'нул back, проверил log file — 20 warning entries после reload плагина.

## How to apply
Для exocortex warnings verification:

```bash
# 1. Counts after specific time (UTC)
awk '$0 >= "[2026-05-17T11:08:00"' /Users/kitelev/vault-2025/exocortex-logs.txt | grep -c "Skipping"

# 2. Unique files affected
awk '$0 >= "[<UTC>"' /Users/kitelev/vault-2025/exocortex-logs.txt | grep "Skipping" | sed 's/.*violation: //' | sort -u

# 3. Recent warnings tail
tail -200 /Users/kitelev/vault-2025/exocortex-logs.txt | grep -iE "WARN|skip|violation"
```

vault-exodev имеет свой log файл `/Users/kitelev/vault-exodev/exocortex-logs.txt` (если открыт там же).
