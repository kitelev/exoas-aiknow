# Frontmatter property + basename fallback pattern для daily/dated assets

**Rule:** When extracting day/date identifier из frontmatter (e.g. `pn__DailyNote_day`, `ems__Effort_day`), **add a fallback that parses YYYY-MM-DD prefix from `file.basename`** before returning null.

**Why:** Frontmatter properties управляются templates/commands. Template change может тихо перестать ставить property; existing files на disk не получают property пока их не обновишь вручную. Empirical: vault-2025 после 2025-10-30 потерял `pn__DailyNote_day` у 217 daily notes — Tasks section пропала, без error log. Basename — durable convention (обязательная для obsidian-calendar-plugin) — robust source of truth.

**Pattern (TS sketch):**

```typescript
const dayProperty = metadata.pn__DailyNote_day;
if (!dayProperty) {
  const basenameDay = file.basename.match(/^(\d{4}-\d{2}-\d{2})/)?.[1];
  if (basenameDay) return { isDailyNote: true, day: basenameDay };
  return { isDailyNote: true, day: null };
}
// ... existing property-driven path
```

**Explicit property takes precedence** when both present (user override capability).

**Where applied:**
- **PR #3170 (v16.7.3, 2026-05-17)** — `DailyNoteHelpers.extractDailyNoteInfo` для `pn__DailyNote_day`.

**How to apply elsewhere:**

- **Any helper extracting `day`/`date`/`week`/`month` от dated asset** — check basename pattern parallel.
- **Tests:** unit case "property missing + basename matches" → returns derived day; "property missing + basename non-matching" → returns null; "both present" → property wins.
- **Не extending to general identity** — basename ≠ frontmatter UUID. Identity всегда через `exo__Asset_uid`.

**Anti-pattern:** auto-backfill missing property из basename при file open. Mutates user data without intent — keep extraction pure. Backfill — separate explicit operation (как `backfill-daily-note-day.py` в этой сессии).
