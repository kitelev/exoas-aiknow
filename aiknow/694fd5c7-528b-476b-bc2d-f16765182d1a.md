# Top-level process.env access crashes Obsidian iOS plugin load

**Rule:** В любом модуле `packages/exocortex/` (или другом transitively-bundled в plugin main.js) запрещён top-level доступ к `process.env.*`, `process.cwd()`, `process.platform` без guard `typeof process !== 'undefined' && process.env`.

**Why:** Obsidian iOS — non-Node JS runtime. Глобальный `process` отсутствует. Module evaluation на iOS бросает `ReferenceError: process is not defined` ДО `onload()`, плагин падает с "failed to load plugin". Симптом для пользователя — flood "Skipping file with invariant violation" Notice'ов (если ошибка частичная) + плагин disabled в Community plugins.

**How to apply:**
1. Top-level constants, использующие `process.env`, оборачивать в guard:
   ```ts
   export const FOO = typeof process !== 'undefined' && process.env
     ? `${process.env.HOME ?? '/tmp'}/...`
     : '/tmp/...';
   ```
2. Внутри method bodies — guard не нужен (методы вызываются только в Node-окружении).
3. Паттерн уже использован в `packages/exocortex/src/services/LoggingService.ts:36`.
4. CLI-only код (ValidatorDaemon, ShapeLoader.loadFromVaultFS) всё равно бандлится в плагин через `index.ts` re-export — нельзя assume "это не вызовется, значит безопасно". **Module-level evaluation = всегда выполняется при import.**

**Empirical reference:**
- Регрессия: PR #3062 (commit `cfd31118`, v15.156.0, 2026-05-03) — заменил IIFE+`require('os')` на top-level `process.env.HOME ?? process.env.USERPROFILE`.
- Affected versions: v15.156.0 — v16.6.1 (все release'ы между регрессией и фиксом).
- Fix: PR #3149 (v16.6.2, 2026-05-17).
- Reporter: user, наблюдал на iPhone после BRAT update с v15.37 → v16.4.1.

**Cross-reference:** `verify-before-assert.md` rule — initially я blamed ShapeLoader.ts dynamic imports без bundle-diff verification. Bundle-diff показал v15.155.0 и v15.155.2 byte-identical → bisect был сбит шумом, реальный регресс на 1 коммит позже.
