# Session closure — pn__DailyNote Tasks regression fix (2026-05-17)

## Контекст

User report: после обновления плагина Exocortex (v15.38 → v16.6.3) Tasks секция в Layout `pn__DailyNote` ассетов перестала отображаться на телефоне. Critical блокер — большая часть mobile usability сломана.

## Two-bug story (root cause)

**Bug 1 — UUID-canon class mismatch:**
`DailyNoteHelpers.extractDailyNoteInfo` (line 25-27, packages/obsidian-plugin/src/presentation/renderers/helpers/DailyNoteHelpers.ts) проверял класс только symbolically:
```typescript
c === "[[pn__DailyNote]]" || c === "pn__DailyNote"
```
После RFC-004 bulk UUID-canon migration (2026-05-16) frontmatter daily-notes стал `exo__Instance_class: [[b04e7a3e-...]]` → `isDailyNote=false` → renderer bail → секция отсутствует.

Parallel issue к #3141 (button builder), который uses parallel patch.

**Bug 2 — Missing `pn__DailyNote_day` property:**
210/211 daily notes в vault-2025 без property (последняя с ней — `2025-10-30 Note.md`). Template/command перестали ставить property с ноября 2025. Helper при отсутствии возвращал `day=null` → renderer bail.

## Что сделано

1. **PR #3158 (v16.7.2)** — UUID-canon match добавлен. 2 unit tests. Merged 14:53Z, released 14:57Z.
2. **Vault backfill** — `/Users/kitelev/Developer/backfill-daily-note-day.py --apply` пропатчил 217 файлов (idempotent: 66 уже had property, 217 patched). Sync uploaded.
3. **PR #3170 (v16.7.3)** — basename-fallback (`YYYY-MM-DD*` префикс → day). 4 new unit tests + updated 1 "no-render" test. Merged 15:19Z, released 15:27Z. CI retry для `daily-archive-filter @flaky-track` flake.
4. **E2E verify via computer-control** — 3 прохода: v16.7.2 с property, v16.7.2 на старой ноте, v16.7.3 БЕЗ property. Все — Tasks секция с 6 задачами рендерится.

## Ключевые решения

- **Hotfix А (узкий hardcode UID)** вместо B (dynamic resolve через metadataCache) — zero refactor surface, parallel-style fix к #3141.
- **Both: backfill + plugin fallback** — defense-in-depth. Backfill = immediate; fallback = future-proof.
- **Restore property после fallback test** — explicit > implicit при tie.
