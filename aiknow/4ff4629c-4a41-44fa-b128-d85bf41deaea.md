## Правило

Когда `shacl-pre-validate.sh` возвращает exit=1 для содержимого нового asset → **остановиться и спросить пользователя**. **НЕ** автоматически удалять поля / refs / wikilinks чтобы pass check.

## Why

- "Delete to make problem go away" — anti-pattern, отдельно зафиксирован в global rules
- SHACL violation часто signal реальной проблемы (broken ref, deleted parent, wrong class) — drop'нуть = потерять данные о связи + замаскировать issue
- Если auto-fix без consent → user может потерять важную semantic info (например parent containment в asset hierarchy)

## How to apply

1. Compose asset content
2. Run `shacl-pre-validate.sh` (Gate 0.9)
3. Exit=0 → Write ✓
4. **Exit=1 → STOP**, present violation to user:
   - Что именно не conformant
   - 3-4 варианта решения (drop ref / fix target / recreate stub / leave broken — accept +1 violation)
5. Wait for user choice → apply
6. **Никогда** silent strip ref для cosmetic green-pass

## Антипаттерн

- ❌ SHACL exit=1 → автоматически awk-sed-strip "проблемного" поля → Write → Done
- ✅ SHACL exit=1 → present violation + 4 fix options → wait user → apply
