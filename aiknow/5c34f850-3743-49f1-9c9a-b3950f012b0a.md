# Empirical verify ПОСЛЕ child COMPLETED ping

**Rule:** Когда дочерняя Claude сессия emit'ит `COMPLETED` ping — НЕ передавать пользователю «готово» без empirical re-verify через Bash. Child summary описывает intent, не результат.

**Why:**
- 2026-05-16 первый CLI v16 child объявил `COMPLETED: PR #3129 merged, npm 15.178.0 released, 4 TBox assets pushed`. Звучало завершённо. Reality: scope был «Phase 1a+1b+T2.x+T3.x до v16.0.0» — child выполнил только slot T1.7+T1.8 и закрылся. Без empirical verify (`npm view` = 15.178.0 ≠ 16.0.0; `gh release list` отсутствует v16) я бы сообщил пользователю что цель достигнута.
- 2026-05-16 Phase 1b child объявил `COMPLETED: 3 aliases seeded, 50 cliName+50 destructive backfilled, PR #3131 merged, v16.1.0 released`. Empirical verify подтвердил 50+50 backfill и v16.1.0, но первый `grep -l "find__Alias\"\]\]"` показал 0 instances — ложный negative из-за wrong bracket syntax. Расширенный grep подтвердил 3 instances. Один verify-pass обнаружил ту правильную аномалию.
- Per global rule `verify-before-assert.md` — claims о state не должны браться из memory / summary / child reports. Свежий empirical check в каждой turn.

**How to apply:**
1. **Triggers:** child emit'ит `COMPLETED` ping ИЛИ `EXITED rc=0` ИЛИ delivers summary с specific facts (PR merged, version released, files created, tests passing).
2. **Verify-set construction:** для каждого specific claim в COMPLETED ping — соответствующий Bash check:
   - «PR #X merged» → `gh pr view X --json state,statusCheckRollup`
   - «vY.Z.W published» → `npm view <pkg> version` + `gh release view vY.Z.W`
   - «N files created/modified» → `find <path> -name pattern | wc -l` или `grep -rl pattern | wc -l`
   - «Tests passing» → `npx <pkg> --help` (для CLI) или real-command execution
3. **Sample real usage** — если CLI command claim'ит работать, запусти его на real vault asset (не только `--help`). 2026-05-16: `find --class ems__Task` returned paths (✅); `find --class` без alias-seed → unknown option error (caught gap).
4. **Multi-grep ракурсы** — если ищешь instances в vault через grep, попробуй ≥2 разные patterns (с и без trailing brackets, regex variations). Single grep может дать ложный 0.
5. **Report verify-outcome пользователю** — не «child говорит готово», а «verified empirically: gh PR #X = MERGED, npm version = Y, --help shows commands A,B,C, no L,M absent. Child gap: feature Z not implemented». Honest 100%, не sanitized.

**Когда можно skip empirical re-verify:**
- Read-only research dump (child writes prompt or asset, no code merge, no release) — child может deliver text-only output.
- Child explicitly objavil partial с BLOCKED ping — parent уже знает что не всё done.

**Applies to:**
- Любой spawn-nested-claude COMPLETED ping
- Любой PMBOK child closure
- Любой /github-issue-executor done report
- В целом — любой автономный agent claiming state change
