# UUID-canon class refs require parallel-match clauses in every helper

**Rule:** When writing or auditing code that matches an `exo__Instance_class` value symbolically (e.g. `c === "[[pn__DailyNote]]" || c === "pn__DailyNote"`), **also accept the UID-canon form** (`"[[<uid>]]"` and bare `<uid>`).

**Why:** After RFC-004 UUID-canon bulk rename (2026-05-16 для vault-2025), TBox class files renamed to `<uid>.md`, and instances' `exo__Instance_class` стало `[[<uid>]]` instead of `[[pn__DailyNote]]`. Helpers that only matched symbolic form started returning `false`/`null` silently, причиняя feature regressions без error logs.

**Documented incidents (parallel fixes):**
- **PR #3141 (commit `ac0b3c4a`, 2026-05-16)** — `DynamicCommandButtonGroupBuilder` expanded UUID class refs to symbolic via `app.metadataCache` so CommandBindings match after UUID-canon. **Buttons disappeared after migration.**
- **PR #3158 (v16.7.2, 2026-05-17)** — `DailyNoteHelpers.extractDailyNoteInfo` added narrow hardcode of `pn__DailyNote` UID (`b04e7a3e-6b49-4984-9f8d-b74e9f36818b`) alongside symbolic. **Tasks section disappeared from DailyNote.**

**How to apply:**

1. **При написании нового helper** что проверяет class symbolically — добавить branch для UID. Узкий hardcode (если class известен, как `pn__DailyNote`) или динамический resolve через `app.metadataCache.getFirstLinkpathDest` (как #3141). Tradeoff: hardcode = zero refactor, dynamic = generic but requires API.

2. **При аудите код-базы** — grep:
   ```
   grep -rn '=== "\[\[\(pn\|ems\|ims\|aiknow\|pmbok\)__\|=== "\(pn\|ems\|ims\|aiknow\|pmbok\)__' packages/obsidian-plugin/src packages/exocortex/src | grep -v test
   ```
   Найденные места — candidates для UID parallel-match. Stretch: extract utility `isClassMatch(value, symbolicName, uid)`.

3. **Тестировать обе формы** — `[[symbolic]]`, `[[uid]]`, bare `uid` — unit tests должны cover все три.

**Anti-pattern:** добавить только symbolic match, надеясь что pre-write hook не пустит UID-form в frontmatter. UUID-canon ЭТО default invariant; pre-write hook не предотвращает legitimate UID storage.
